# Product overview

Mincemeat is a platform for managing cloud instances and hosting static
sites. It provides a unified interface for developers and teams to
deploy, monitor, and maintain web applications without managing
infrastructure directly.

## Core features

### Cloud Instances

A cloud instance is a dedicated Linux container assigned to you by an
administrator. You can monitor its status, view resource metrics, open
a web terminal, and manage proxy settings. Instances are ideal for
running applications that need persistent compute resources.

### Static Sites

Static Sites lets you host websites without managing servers. Upload
pre-built files or connect a GitHub repository, and Mincemeat serves
your content on a shared subdomain or your own custom domain. Every
deployment is immutable, so you can roll back to any previous version
instantly.

### Domains

Connect custom domains to your instances or static sites. Mincemeat
handles SSL certificates automatically and provides DNS validation
guidance so you can point your domain to the right endpoint.

### Account management

Manage your account security with password changes, two-factor
authentication, recovery codes, and an audit history of your account
activity.

## Who uses Mincemeat

Mincemeat is designed for two primary roles:

| Role | What you can do |
| --- | --- |
| User | View assigned instances, deploy static sites, connect domains, manage your account settings. |
| Administrator | Create and assign instances, manage organisation-wide settings, control resource limits, and manage user access. |

This documentation is written for the **user** role. If you need
administrator guidance, check with your organisation's Mincemeat admin.

## How the platform works

Mincemeat abstracts infrastructure management behind a web interface.
You do not need to provision servers, configure load balancers, or
manage SSL certificates. Instead, you:

1. Sign in to your account.
2. Access your assigned resources (instances or static sites).
3. Deploy or configure your applications.
4. Monitor status and make adjustments as needed.

## What is not covered here

This documentation does not cover:

- Administrator tasks like creating instances or managing users.
- Infrastructure setup, server configuration, or database administration.
- Pricing, billing, or service-level agreements.
- Internal operational runbooks or deployment internals.
